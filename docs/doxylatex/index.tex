The primary design goals are\+:


\begin{DoxyItemize}
\item easy to use for the end users\+: we use multi-\/threading and synchronous IO to support concurrency so you only need to focus on the logic. We carefully craft the library to use computing resources efficiently so performance should never be an issue.
\item easy to reason about the code\+: we use the most native data structures, the simplest algorithms, and intuitive interfaces.
\item easy to debug (networking and logic) errors\+: extensive assertion and logging facilities.
\item superior reliability
\end{DoxyItemize}\hypertarget{index_autotoc_md15}{}\doxysubsection{Implementation}\label{index_autotoc_md15}
Orca is implemented in plain C. The symbols are organized in a intuitive fashion to be easily identified by matching to the R\+E\+ST A\+PI docs.

There are three purposes for this\+:


\begin{DoxyEnumerate}
\item Reduce the need of thoroughly documenting every Orca A\+PI
\item Reduce our user\textquotesingle{}s cognitive burden of having to read both Orca A\+PI documenation and supported R\+E\+ST A\+PI documentations.
\item The codebase becomes easier to navigate.
\end{DoxyEnumerate}

Orca\textquotesingle{}s implementation has minimum external dependencies to make bot deployment deadly simple.
